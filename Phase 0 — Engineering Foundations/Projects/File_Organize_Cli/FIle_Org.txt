# FileOrg CLI Project

Goal:
Build a command line tool named `fileorg` that helps organize files, clean junk,
find duplicates, archive old files, show large files, create backups, schedule
jobs, and undo the last operation.

Main idea:
The user runs commands like:

fileorg organize ~/Downloads
fileorg watch ~/Downloads
fileorg duplicates ~/Documents
fileorg cleanup
fileorg archive ~/Downloads --older-than 30
fileorg large --top 10
fileorg backup ~/Projects /mnt/backup
fileorg schedule organize ~/Downloads --every 5m
fileorg undo

This project teaches:
- Python file handling
- CLI argument parsing
- Folder traversal
- Moving, copying, deleting, and archiving files
- Hashing files to detect duplicates
- JSON config and history files
- Background watching/scheduling basics
- Safe undo system
- Testing a real utility


1. Project Features

1. organize

Command:
fileorg organize ~/Downloads

Purpose:
Organize files in a folder by file type.

Example:
Before:
~/Downloads/photo.png
~/Downloads/song.mp3
~/Downloads/resume.pdf
~/Downloads/app.zip

After:
~/Downloads/Images/photo.png
~/Downloads/Audio/song.mp3
~/Downloads/Documents/resume.pdf
~/Downloads/Archives/app.zip

Common categories:
- Images: .jpg, .jpeg, .png, .gif, .webp, .svg
- Documents: .pdf, .doc, .docx, .txt, .md, .pptx, .xlsx
- Videos: .mp4, .mkv, .mov, .avi
- Audio: .mp3, .wav, .aac, .flac
- Archives: .zip, .rar, .tar, .gz, .7z
- Code: .py, .js, .html, .css, .java, .cpp
- Others: everything else

Implementation steps:
1. Take folder path from command line.
2. Convert `~` to full path using `Path(...).expanduser()`.
3. Check if folder exists.
4. Loop through all files in that folder.
5. Skip folders.
6. Read each file extension.
7. Decide category from extension.
8. Create category folder if it does not exist.
9. Move file into category folder.
10. If file name already exists, rename safely.
11. Save every move operation in history for undo.

Important safety rule:
Never overwrite a file. If `photo.png` already exists, create:
photo_1.png
photo_2.png
photo_3.png


2. watch

Command:
fileorg watch ~/Downloads

Purpose:
Keep watching a folder and automatically organize new files.

Example:
If the user downloads `invoice.pdf`, FileOrg automatically moves it to:
~/Downloads/Documents/invoice.pdf

Implementation steps:
1. Start a long-running process.
2. Watch the target folder for new files.
3. When a new file appears, wait a few seconds so download completes.
4. Run the same logic as `organize`.
5. Print what was moved.

Simple beginner version:
Use a loop:
- Scan folder every 5 seconds.
- Remember files already seen.
- If a new file appears, organize it.

Advanced version:
Use the `watchdog` Python package.

Command example:
pip install watchdog


3. duplicates

Command:
fileorg duplicates ~/Documents

Purpose:
Find duplicate files in a folder.

Duplicate meaning:
Two files are duplicates if their content is exactly the same.
Do not only compare file names because two files can have different names but
same content.

Implementation steps:
1. Take folder path from command line.
2. Walk through all files inside folder and subfolders.
3. For each file, calculate a hash.
4. Use SHA256 hash.
5. Store hash in dictionary.
6. If same hash appears more than once, show duplicate group.

Example output:
Duplicate group 1:
- ~/Documents/report.pdf
- ~/Documents/old/report-copy.pdf

Duplicate group 2:
- ~/Documents/photo.png
- ~/Documents/backup/photo.png

Optional delete feature for later:
fileorg duplicates ~/Documents --delete

Safety rule:
At first, only show duplicates. Do not delete automatically.


4. cleanup

Command:
fileorg cleanup

Purpose:
Clean common temporary files from system or project folders.

Examples of cleanup targets:
- __pycache__ folders
- .pyc files
- .DS_Store files
- Thumbs.db files
- temporary files ending with .tmp
- empty folders

Implementation steps:
1. Decide default cleanup location.
2. Safer beginner choice: clean current directory only.
3. Walk through files and folders.
4. Find cleanup patterns.
5. Show what will be removed.
6. Ask confirmation before deleting.
7. Delete selected files/folders.
8. Save deleted file info in history.

Better command later:
fileorg cleanup ~/Projects

Safety rule:
Do not delete user documents, images, videos, code, or downloads.
Only delete known junk files.


5. archive

Command:
fileorg archive ~/Downloads --older-than 30

Purpose:
Move old files into an archive folder.

Meaning:
`--older-than 30` means files older than 30 days.

Example:
~/Downloads/old-file.pdf

Moves to:
~/Downloads/Archive/2026-06/old-file.pdf

Implementation steps:
1. Take folder path.
2. Take number of days from `--older-than`.
3. Get current date/time.
4. Check each file modified time.
5. If file age is greater than given days, move to Archive folder.
6. Create archive month folder like `Archive/YYYY-MM`.
7. Save move operation in history.

Safety rule:
Skip files already inside Archive folder.


6. large

Command:
fileorg large --top 10

Purpose:
Show the biggest files.

Default behavior:
Scan current directory.

Better command later:
fileorg large ~/Downloads --top 10

Implementation steps:
1. Take optional folder path.
2. Take number from `--top`.
3. Walk through folder.
4. Get file size using `stat()`.
5. Sort files by size descending.
6. Print top N files with readable size.

Example output:
1. 2.1 GB  ~/Videos/movie.mp4
2. 800 MB  ~/Downloads/ubuntu.iso
3. 120 MB  ~/Documents/data.csv

Helper function:
Create `format_size(bytes)` to convert bytes into KB, MB, GB.


7. backup

Command:
fileorg backup ~/Projects /mnt/backup

Purpose:
Copy one folder to another location as a backup.

Example:
Source:
~/Projects

Destination:
/mnt/backup

Result:
/mnt/backup/Projects_backup_2026-06-06_10-30

Implementation steps:
1. Take source path.
2. Take destination path.
3. Check source exists.
4. Check destination exists or create it.
5. Create backup folder with date/time.
6. Copy all files and folders.
7. Print total files copied.
8. Save backup operation in history.

Python tool:
Use `shutil.copytree()`.

Safety rule:
Do not overwrite an existing backup folder.


8. schedule

Command:
fileorg schedule organize ~/Downloads --every 5m

Purpose:
Run a FileOrg command repeatedly.

Example:
Every 5 minutes, organize Downloads folder.

Supported time formats:
- 5s = every 5 seconds
- 5m = every 5 minutes
- 2h = every 2 hours

Implementation steps:
1. Parse the command after `schedule`.
2. Parse time from `--every`.
3. Convert time to seconds.
4. Start loop.
5. Run selected command.
6. Sleep for given time.
7. Repeat until user stops with Ctrl + C.

Beginner version:
Only support:
fileorg schedule organize PATH --every TIME

Advanced version:
Support duplicates, cleanup, archive, backup also.


9. undo

Command:
fileorg undo

Purpose:
Undo the last operation.

Example:
If `organize` moved:
~/Downloads/photo.png -> ~/Downloads/Images/photo.png

Then `fileorg undo` moves it back:
~/Downloads/Images/photo.png -> ~/Downloads/photo.png

Implementation steps:
1. Every command that changes files must save history.
2. History file stores operations in JSON.
3. `undo` reads the last operation.
4. Reverse each action in opposite order.
5. Remove the undone operation from history.

History example:
[
  {
    "command": "organize",
    "time": "2026-06-06 10:30:00",
    "actions": [
      {
        "type": "move",
        "source": "/home/yash/Downloads/photo.png",
        "destination": "/home/yash/Downloads/Images/photo.png"
      }
    ]
  }
]

Undo move:
Move destination back to source.

Undo backup:
Ask confirmation, then remove backup folder.

Undo cleanup:
Harder because deleted files are gone.
Better approach: cleanup should move files to a FileOrg trash folder first.

Main commands:

fileorg large [PATH] --top NUMBER

12. Data Storage

Store FileOrg data in user home folder:

~/.fileorg/
  config.json
  history.json
  trash/

config.json:
Stores categories and extensions.

history.json:
Stores previous file-changing operations.

trash/:
Stores cleanup files instead of deleting immediately.

14. Important Helper Functions

resolve_path(path):
- Expands `~`.
- Converts relative path to absolute path.
- Checks if path exists.

safe_move(source, destination_folder):
- Moves a file.
- Avoids overwriting.
- Returns final destination path.

get_category(file_path):
- Reads extension.
- Returns category name.

hash_file(file_path):
- Reads file in chunks.
- Returns SHA256 hash.

format_size(size_in_bytes):
- Converts bytes to KB, MB, or GB.

save_history(operation):
- Adds operation to history.json.

load_history():
- Reads history.json.

undo_last():
- Reverses last operation.

parse_interval("5m"):
- Converts 5m to 300 seconds.


15. Safety Checklist

Before moving/deleting/copying files:
- Check path exists.
- Check path is a file or folder as expected.
- Never overwrite existing files.
- Skip hidden/system files in beginner version.
- Print clear messages.
- Save history before or immediately after actions.
- For cleanup, move to trash instead of permanent delete.
- For backup, never overwrite backup folders.

20. Final Project Summary

FileOrg is a real-world Python CLI project. It starts simple by organizing files
into folders, then becomes more powerful with duplicates, archive, cleanup,
backup, watch, schedule, and undo features.

The most important part is safety. A file organization tool must not overwrite
or delete user files by accident. That is why this project needs safe move,
history, trash, confirmation, and undo.
