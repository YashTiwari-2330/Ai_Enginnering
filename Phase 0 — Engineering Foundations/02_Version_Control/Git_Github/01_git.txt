# Git and GitHub Basics

This file explains the basic ideas of Git, GitHub, repositories, commits, and
branches. It also includes the most common commands used when starting a new
project.

---

## 1. What is Git?

Git is a version control system.

It helps you:

- Track changes in your files.
- Save different versions of your project.
- Go back to an older version if something breaks.
- Work with other developers without losing code.

Example:

```bash
git status
git add index.html
git commit -m "Add home page"
```

---

## 2. What is GitHub?

GitHub is an online platform where you can store Git repositories.

Git is the tool on your computer.
GitHub is the website where you can upload and share your Git project.

GitHub helps you:

- Store your code online.
- Share your project with others.
- Work with a team.
- Review code using pull requests.
- Track issues and project tasks.

---

## 3. What is a Repository?

A repository, often called a repo, is a project folder tracked by Git.

It contains:

- Your project files.
- Git history.
- Branches.
- Commits.
- Tags.

Create a Git repository:

```bash
git init
```

Check repository status:

```bash
git status
```

---

## 4. What is a Commit?

A commit is a saved snapshot of your project.

Think of it like a checkpoint in your work. If something goes wrong later, you
can return to an older commit.

Basic commit workflow:

```bash
git status
git add .
git commit -m "Write a clear commit message"
```

Command meaning:

- `git status` shows changed files.
- `git add .` stages all changed files.
- `git commit -m "message"` saves the staged changes.

Good commit message examples:

```bash
git commit -m "Add login form"
git commit -m "Fix navbar layout"
git commit -m "Update README instructions"
```

---

## 5. What is a Branch?

A branch is a separate line of development.

Branches allow you to work on a feature or bug fix without directly changing
the main project code.

Example:

```bash
git switch -c feature-login
```

This creates and switches to a new branch named `feature-login`.

Common branch commands:

```bash
git branch
git branch branch_name
git switch branch_name
git switch -c branch_name
git checkout branch_name
```

Command meaning:

- `git branch` shows all local branches.
- `git branch branch_name` creates a new branch.
- `git switch branch_name` switches to an existing branch.
- `git switch -c branch_name` creates and switches to a new branch.
- `git checkout branch_name` is the older command for switching branches.

Recommended modern command:

```bash
git switch branch_name
```

---

## 6. Connect Local Git Project to GitHub

Step 1: Create a new repository on GitHub.

Step 2: Copy the repository URL.

Example URL:

```text
https://github.com/username/repository-name.git
```

Step 3: Run these commands in your local project folder:

```bash
git init
git add .
git commit -m "Initial commit"
git branch -M main
git remote add origin https://github.com/username/repository-name.git
git push -u origin main
```

Command meaning:

- `git init` starts Git tracking.
- `git add .` stages all files.
- `git commit -m "Initial commit"` creates the first commit.
- `git branch -M main` renames the current branch to `main`.
- `git remote add origin URL` connects local Git to GitHub.
- `git push -u origin main` uploads the `main` branch to GitHub.

---

## 7. Clone an Existing GitHub Repository

If a repository already exists on GitHub, clone it:

```bash
git clone https://github.com/username/repository-name.git
```

Then move into the project folder:

```bash
cd repository-name
```

---

## 8. Pull and Push

Use `pull` to download latest changes from GitHub:

```bash
git pull origin main
```

Use `push` to upload your commits to GitHub:

```bash
git push origin main
```

If the branch is already tracking GitHub, you can usually use:

```bash
git pull
git push
```

---

## 9. Useful Daily Git Commands

```bash
git status
git log --oneline
git branch
git switch branch_name
git add .
git commit -m "message"
git pull
git push
```

Command meaning:

- `git status` checks current changes.
- `git log --oneline` shows commit history in short format.
- `git branch` lists branches.
- `git switch branch_name` changes branch.
- `git add .` stages all changed files.
- `git commit -m "message"` saves changes.
- `git pull` gets latest remote changes.
- `git push` sends local commits to remote.

---

## 10. Common Mistakes

Mistake:

```bash
git commit "message"
```

Correct:

```bash
git commit -m "message"
```

Mistake:

```bash
git add origin URL
```

Correct:

```bash
git remote add origin URL
```

Mistake:

```bash
git push -u main
```

Correct:

```bash
git push -u origin main
```

---

## 11. Simple Git Workflow

```bash
git status
git add .
git commit -m "Explain what changed"
git push
```

This is the most common workflow you will use every day.
